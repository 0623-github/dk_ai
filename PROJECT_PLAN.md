## 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| Ollama 性能瓶颈 | 高 | 支持远程 Ollama / 多实例部署 |
| 数据库迁移复杂 | 中 | 使用 GORM / 编写迁移脚本 |
| 前端状态混乱 | 中 | 引入状态管理库 |
| 部署环境差异 | 低 | Docker 统一环境 |

---

## 需要老大决策的事项

1. **数据库选型**: SQLite (简单) vs PostgreSQL (扩展性强)
2. **部署目标**: 本地运行 vs 云服务器 vs 两者都要
3. **用户系统**: 是否需要多用户支持，还是单机版即可
4. **模型策略**: 只用 Ollama，还是也支持 OpenAI/Claude API
