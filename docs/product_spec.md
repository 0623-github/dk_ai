## 1. 产品愿景

让小米小爱音箱接入本地/云端大模型，实现更智能、更自然的语音对话体验。

**核心价值**:
- 🎯 突破小爱音箱原有技能限制，接入任意大模型
- 🔒 支持本地部署（Ollama），保护隐私
- 🌐 同时支持远程 API（OpenAI/Claude等）
- 💬 保留小爱原有的音色和唤醒体验

## 4. 技术方案

### 4.1 小爱音箱接入方案

调研两种方案：

**方案A: MiIO 协议模拟（推荐）**
- 模拟小米云端服务，拦截音箱请求
- 优点：无需刷机，保留原有功能
- 缺点：需要局域网内运行服务

**方案B: 刷机替换固件**
- 刷入自定义固件，直接对接 API
- 优点：完全控制，可离线运行
- 缺点：操作复杂，可能变砖

**选定方案A**，先实现局域网接入。

### 4.2 语音链路

```
用户语音 → 音箱本地ASR → 文本 → dk_ai服务 → 大模型
                                              ↓
用户收听 ← 音箱播放 ← TTS合成 ← AI回复文本 ←─┘
```
